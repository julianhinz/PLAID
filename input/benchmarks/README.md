# Benchmark Data for Indicator Validation

External benchmark datasets used to validate LLM-generated indicators. Processed CSVs are generated by `code/common/prepare_benchmarks.R` from raw files in `raw/`.

## Preparation

```bash
Rscript code/common/prepare_benchmarks.R
```

This reads `raw/*.xls(x)` and produces the four CSV files below.

## Files

### `bec_hs2012_concordance.csv`
- **Source:** UN Statistics Division, BEC Rev.5 to HS 2012 native concordance
- **URL:** http://unstats.un.org/unsd/trade/classifications/BEC5Correlations.zip
- **Raw file:** `raw/HS2012-17-BEC5 -- 08 Nov 2018.xlsx` (sheet `HS12BEC5`)
- **Columns:** `hs6_code`, `bec_category` (capital / intermediate / consumption)
- **Notes:** HS 2012 is the native vintage against which BEC Rev.5 was defined, so this is the benchmark used for the primary BEC validation (§3.2 of the paper). For HS codes mapped to multiple BEC categories (e.g. INT/CONS), the first listed category is used as primary. 5,203 HS6 codes covered.
- **Accessed:** 2026-04-02

### `bec_hs2017_concordance.csv`
- **Source:** UN Statistics Division, BEC Rev.5 to HS 2017 concordance
- **URL:** http://unstats.un.org/unsd/trade/classifications/BEC5Correlations.zip
- **Raw file:** `raw/HS2012-17-BEC5 -- 08 Nov 2018.xlsx` (sheet `HS17BEC5`)
- **Columns:** `hs6_code`, `bec_category` (capital / intermediate / consumption)
- **Notes:** Derived concordance for HS 2017, used in the cross-revision robustness check (§3.2). For HS codes mapped to multiple BEC categories, the first listed category is used as primary. 5,385 HS6 codes covered.
- **Accessed:** 2026-04-02

### `oecd_ict_hs.csv`
- **Source:** OECD/UNCTAD Definition of ICT Goods, HS 2017 revision
- **URL:** https://unctadstat.unctad.org/EN/Classifications/DimHS2017Products_Ict_Hierarchy.xls
- **Raw file:** `raw/ict_hs2017.xls`
- **Columns:** `hs6_code`, `ict_good` (TRUE/FALSE)
- **Notes:** 92 HS6 codes classified as ICT goods across 5 sub-categories (computers, communication, consumer electronics, electronic components, miscellaneous). This is narrower than our `microchip_content` indicator, which also covers goods with embedded semiconductors (vehicles, appliances, medical devices).
- **Accessed:** 2026-04-02

### `eu_conflict_minerals_hs.csv`
- **Source:** EU Conflict Minerals Regulation (2017/821), Annex I
- **URL:** https://eur-lex.europa.eu/eli/reg/2017/821/oj/eng
- **Raw file:** none (hardcoded from regulation text)
- **Columns:** `hs6_code`, `mineral` (tin/tantalum/tungsten/gold), `regulated` (TRUE)
- **Notes:** 22 HS6 codes covering regulated 3TG ores, concentrates, and metals. CN codes with `ex` prefix (partial coverage) are included at the HS6 level. This is a precise, small list; the LLM classification is expected to flag additional derivative products (alloys, electronics) not in this annex.
- **Accessed:** 2026-04-02
